# 小程序名片体验设计说明

版本：v1.0  
日期：2026-07-06  
用途：交给 Claude / 设计工具生成小程序页面原型，也作为产品体验验收口径。  
定位：本产品是企业电子名片 SaaS，优先服务「员工分发名片、访客了解与联系、企业统一品牌内容」。仅借鉴同类名片工具的版式结构与内容组织方式，视觉风格完全自定，不参考任何第三方产品的具体视觉设计。

---

## 1. 设计目标

小程序要从当前的基础名片骨架升级为「可展示、可编辑、可分发、可追踪」的企业数字名片体验。

目标体验：

- 员工打开后，能快速看到自己的名片、状态、访客数据和分发入口。
- 访客打开后，不只看到员工联系方式，还能看到企业介绍、视频、荣誉图片轮播等内容，形成可信的企业展示页。
- 企业侧能统一模板、品牌、公司介绍、荣誉和团队成员，不让每个员工随意破坏品牌一致性。
- 分享链路只使用 `public_id` + `share_id`，不暴露内部 ID。
- 访客查看名片、保存电话、拨打电话不强制授权。

---

## 2. 信息架构

底部主导航建议保留三栏：

| Tab | 页面名 | 目标 |
|-----|--------|------|
| 首页 | 我的名片 | 员工自己的名片工作台、分发入口、访客数据 |
| 名片夹 | 名片夹 | 客户/交换名片沉淀，后续阶段增强 |
| 企业名片 | 企业名片 | 企业模板、团队成员、企业内容管理入口 |

MVP 可以先实现「首页 + 访客名片详情」，但视觉设计要预留三栏导航。

## 3. 设计 Tokens / 视觉规范

本节是全局基线，所有页面必须严格复用，禁止各页自行发挥导致视觉不一致。数值为建议值，可整体微调，但一经确定全局统一。

### 3.1 色彩

- 品牌主蓝（主按钮、主链接、选中态）：`#2B6CFF`
  - 按下态：`#1E56D6`
  - 禁用态：`#A9C4FF`
  - 浅色底（选中背景、蓝色提示条）：`#EAF1FF`
- 语义色
  - 成功 / 在职认证：`#17B26A`，浅底 `#E7F7EF`
  - 警告 / 试用即将过期：`#F79009`，浅底 `#FEF3E5`
  - 危险 / 已过期已停用：`#F04438`，浅底 `#FDECEC`
- 中性色
  - 主文本：`#1A1D1F`
  - 次文本（职位、辅助说明）：`#6B7280`
  - 占位 / 弱提示：`#9AA1AC`
  - 分割线 / 描边：`#EDEFF2`
  - 卡片背景：`#FFFFFF`
  - 页面背景：`#F4F6F8`

### 3.2 字号与字重（单位 px，行高 1.4–1.5）

- 名片卡姓名：22 / Bold
- 页面标题（导航中间）：18 / Medium
- 卡片区块标题（如「公司介绍」）：16 / Medium
- 正文 / 表单值：14 / Regular
- 辅助说明 / 时间 / 计数：12 / Regular
- 按钮文字：16 / Medium

### 3.3 间距与栅格

- 基础栅格：8pt（可用 4 / 8 / 12 / 16 / 20 / 24）
- 页面左右安全边距：16
- 卡片内边距：16
- 卡片之间纵向间距：12
- 区块标题与内容间距：12

### 3.4 圆角与阴影

- 名片卡 / 内容卡圆角：12
- 按钮圆角：10
- 标签 / 徽章圆角：6
- 头像：圆形
- 卡片阴影：`0 2px 8px rgba(16,24,40,0.06)`（轻，不要重投影）

### 3.5 关键组件规格

- 名片卡：横版，比例 `1.586 : 1`（标准名片 85.6×54），宽度=屏宽−32，圆角 12，右下角预留状态角标位。
- 主按钮「发名片」：高 48，蓝底白字，圆角 10，占据主操作区宽度，视觉权重明显高于其他入口。
- 次级按钮 / 工具入口：图标 24（线性描边 1.5）+ 12px 文字，横向等分排列。
- 状态提示条：整条浅色语义底 + 同语义深色文字，高度自适应，位于名片卡上方，不遮挡卡片。
- 锁定字段：字段值置灰（`#9AA1AC`）+ 右侧小锁图标 + 尾部说明「企业统一维护」，点击不弹编辑，仅弹一次说明提示。
- 底部固定操作栏：白底 + 顶部 1px 分割线 `#EDEFF2`，安全区适配，主操作用主蓝。

### 3.6 状态设计（每个数据页都要覆盖）

- 加载中：骨架屏或居中 loading，不留白屏。
- 空状态：插画/图标 + 一句说明 + 可选主操作（如名片夹空态引导「拍名片」）。
- 网络错误：错误说明 + 「重试」按钮。
- 失效态：名片过期/停用时，分发类操作禁用并给出文案。

## 4. 员工首页：我的名片

页面标题：我的名片

核心目标：员工一进来就知道「我的名片是什么样、能不能发、最近有没有人看、下一步怎么发」。

页面模块：

1. 顶部导航
   - 左侧菜单图标
   - 中间标题「我的名片」
   - 右侧微信小程序系统胶囊区

2. 状态提醒
   - 试用过期 / 企业名片过期 / 名片已停用 / 信息未完善
   - 用浅色提示条展示，不遮挡主操作
   - 状态文案要直接，例如「您的企业名片已过期，请及时续费」

3. 当前名片预览
   - 横版卡片
   - 支持企业 Logo
   - 支持员工姓名、职位、手机号、公司、地址、座机、邮箱
   - 支持自定义背景图、品牌色、版式
   - 卡片右下角显示状态，例如「名片已失效」

4. 编辑快捷栏
   - 编辑资料
   - 名片样式
   - 切换名片

5. 主分发按钮
   - 大按钮「发名片」
   - 蓝色主按钮，保持高优先级

6. 工具入口
   - 名片图片
   - 文字转名片
   - 名片榜单
   - 名片码
   - 全部

7. 交换请求卡片
   - 展示「有 1 人请求与您交换名片」
   - 请求人头像、姓名、身份、公司
   - 操作：忽略、同意

8. 我的名片数据
   - 我的访客
   - 我看过的
   - 好友新名片
   - 最近访客列表：头像、姓名、职位、公司、访问次数、交换状态、访问时间

设计重点：

- 首页不是表单页，而是员工工作台。
- 名片卡必须是首屏视觉核心。
- 「发名片」必须比其他入口更显眼。
- 过期、停用、试用状态要能被看到，但不要把核心内容压没。

## 5. 发名片页

触发方式：首页主按钮「发名片」。形态建议为底部弹层（bottom sheet），也可做独立页；优先弹层，减少跳转。

页面目标：让员工用最少步骤把名片发出去，并沉淀分发数据。

模块：

1. 名片缩略预览
   - 顶部展示当前名片小卡（姓名、职位、公司、企业 Logo），确认发的是哪张。

2. 分发方式
   - 转发给微信好友 / 群（触发小程序转发）
   - 生成名片海报：名片卡 + 企业 Logo + 小程序码 + 一句 slogan，可保存到相册
   - 名片小程序码：单独展示可截图的码
   - 复制链接

3. 底部说明
   - 一句轻提示：分享链接仅使用 `public_id` + `share_id`，不暴露内部信息。

交互与规则：

- 分享链路只使用 `public_id` + `share_id`，不拼接内部 ID。
- 名片处于失效 / 过期 / 停用状态时，分发方式禁用并提示「名片已失效，请先续费/激活」。
- 每种分发方式记录渠道埋点（转发 / 海报 / 码 / 链接）。
- 海报图片生成走既有图片规则（大小、格式校验）。

设计要求：

- 弹层高度不超过屏幕 3/4，内容不足时自适应收缩。
- 分发方式用图标 + 文字横向或宫格排列，主推「转发」和「海报」。
- 海报预览要按真实名片比例展示，保存按钮用主蓝。

## 6. 员工编辑页

页面标题：编辑资料

页面目标：让员工编辑被企业允许编辑的个人字段。

字段分组：

1. 基础信息
   - 头像
   - 姓名
   - 职位
   - 部门
   - 个人简介

2. 联系方式
   - 手机
   - 座机
   - 邮箱
   - 微信号
   - 企业微信入口
   - 地址
   - 官网

3. 展示标签
   - 专长标签
   - 产品标签
   - 服务标签

4. 隐私设置
   - 手机是否公开
   - 邮箱是否公开
   - 微信是否公开
   - 地址是否公开
   - 企业微信是否展示

5. 企业锁定字段
   - 如果企业不允许员工编辑某字段，字段置灰，并显示「企业统一维护」

交互要求：

- 保存后触发公开名片缓存失效。
- 手机号等敏感字段默认不公开。
- 不能让员工打开企业禁用的字段。

## 7. 名片样式页

页面标题：名片样式

页面目标：员工可在企业允许的范围内选择名片视觉。

功能：

- 选择模板：横版商务、极简、品牌图、深色、活动版
- 选择背景图：企业预设背景、个人上传背景
- 选择品牌色：若企业锁定品牌色，则员工不可改
- 实时预览名片卡
- 应用模板

设计要求：

- 模板卡片以真实名片比例展示。
- 员工只能选择本企业可用模板。
- 所有模板必须适配访客详情页首屏。

## 8. 访客名片详情页

页面标题：员工姓名或「名片详情」

页面目标：访客打开分享后，快速建立对员工和企业的信任，并完成联系动作。

页面结构：

1. 员工名片首屏
   - 企业自定义背景图
   - 企业 Logo
   - 姓名
   - 职位
   - 公司名称
   - 地址
   - 手机
   - 座机
   - 邮箱
   - 名片状态
   - 自定义版式

2. 访客互动区
   - 几人看过
   - 点赞 / 靠谱
   - 我的认证：在职、实名、官方认证等

3. 联系动作区
   - 电话
   - 邮箱
   - 地址
   - 微信 / 企业微信
   - 纸质名片
   - 保存到通讯录
   - 转发名片

4. 公司介绍
   - 标题「公司介绍」
   - 富文本段落
   - 支持图片插入
   - 支持展开全文

5. 企业视频
   - 视频封面
   - 播放按钮
   - 支持全屏播放
   - 可展示一个主视频，后续支持多个视频

6. 公司荣誉
   - 标题「公司荣誉」
   - 荣誉文字介绍
   - 荣誉图片左右轮播
   - 图片支持点击预览大图
   - 每张荣誉图片可配置标题和说明

7. 底部固定操作
   - 访客视角：保存电话、拨打电话、加企微、转发
   - 员工预览视角：编辑、个人信息、名片样式、隐私设置、发名片

重要要求：

- 访客查看名片不强制授权。
- 访客看到的手机号、邮箱、微信等字段必须经过企业规则和员工隐私双重判定。
- 公司介绍、视频、荣誉属于企业内容，员工可引用，默认由企业管理员维护。
- 公开详情页 GET 内容可缓存，不返回 visit_token；访问 token 由 POST `/visit` 获取。

## 11. 内容编辑能力

企业内容建议由后台 Web 完整维护，小程序可提供轻量入口。

需要支持：

- 公司介绍：标题、正文、图片、排序、是否展示
- 企业视频：视频 URL / COS 文件、封面图、标题、排序、是否展示
- 公司荣誉：标题、正文、荣誉图片数组、图片说明、排序、是否展示
- 名片模板：背景图、Logo、颜色、布局 JSON、适用部门、是否默认

审核与保障：

- 图片、视频上传后做大小、类型、格式校验。
- 图片去 EXIF，视频限制大小和时长。
- 外链只允许 `https://` 和企业白名单域名。
- 内容更新后触发公开名片缓存失效。

## 13. 给 Claude 的设计提示词

请基于这份说明设计一套微信小程序页面。页面的版式结构与内容模块以本文档为准，视觉风格请独立设计一套清爽、可信的企业级商业工具界面，不参考或照搬任何第三方名片产品的品牌、配色、图标或样式。页面要适合企业电子名片 SaaS，重点体现员工名片展示、企业内容背书和分发转化。

需要产出页面：

1. 我的名片首页
2. 发名片页（分享弹层）
3. 编辑资料页
4. 名片样式页
5. 访客名片详情页
6. 企业名片页
7. 名片夹页

访客名片详情页必须包含：

- 顶部员工名片自定义背景和版式
- 公司介绍图文
- 企业视频
- 公司荣誉图文
- 荣誉图片左右轮播
- 联系动作区
- 底部固定主操作

视觉要求：

- 微信小程序尺寸优先，移动端 375px 宽设计。
- 严格使用第 3 节「设计 Tokens / 视觉规范」的色彩、字号、间距、圆角与组件规格，所有页面保持一致，不要各页自定义配色。
- 保持白底、浅灰页面背景、蓝色主按钮、清晰卡片分区。
- 每个数据页都要覆盖加载 / 空 / 错误 / 失效状态（见 3.6）。
- 不要做营销落地页，要做真实可用的小程序工具界面。
- 所有文字和按钮在小屏下不能拥挤或重叠。

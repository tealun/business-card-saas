# Handoff：智云名片管理后台（系统后台 + 企业后台）+ 登录页

## Overview

本次设计交付两大部分：

1. **管理后台原型**（`Admin Backoffice.dc.html`）—— 覆盖「系统管理后台」与「企业管理后台」两套后台的信息架构、页面布局、核心交互（侧边栏导航、抽屉编辑、二次确认弹窗、表单校验、Tab 切换等）。
2. **登录页**（`Admin Login.dc.html`）—— 高端科技感的后台登录页，含动态背景（互联网状节点）与矢量插画（云、名片、纸飞机、双人像、增长图表）。

设计以 `original-architecture-guide.md`（产品侧提供的《管理后台界面架构设计文档》）为基线展开，本 README 是对该文档在**页面实现、组件划分、交互细节**上的补充说明，请两份文档对照阅读。

## About the Design Files

`Admin Backoffice.dc.html` 和 `Admin Login.dc.html` 是**设计参考原型**（HTML + 内联样式实现的可交互 Demo），不是可以直接复制上线的生产代码。请按 `original-architecture-guide.md` 第 11 节推荐的技术栈（React + TypeScript + Vite + Ant Design 或同等组件库 + TanStack Query）在实际项目中重新实现，保留本设计确定的布局结构、视觉规范与交互行为。

打开方式：用浏览器直接打开 `Admin Backoffice.dc.html` / `Admin Login.dc.html` 即可看到可点击的原型（依赖同目录下的 `support.js`，请勿单独拆分该文件）。

## Fidelity

**高保真（hifi）**：视觉样式（色彩、字号、圆角、间距、状态标签、抽屉/弹窗规范）已确定，请按此还原。

**页面中的具体数据（企业名称、成员姓名、金额、时间等）均为示意占位数据，不代表真实字段值或真实业务数字，请开发者按后端真实接口自行对接和填充。** 需要还原的是：字段结构（表格有哪些列、详情有哪些分区）、组件类型（标签/开关/表格/图表）、交互行为与状态流转，而不是具体数值。

## 整体信息架构

后台分两套导航（登录后按账号类型进入其一，不支持前端自由切换身份 —— 原型里的顶部/侧边「系统后台/企业后台」切换按钮**仅用于演示两套设计**，实际产品不应存在这个开关）：

### 系统管理后台导航
总览 / 企业（企业授权中心）/ 授权与回调 / 商业化 / 功能开关 / 运维 / 审计 / 系统账号

### 企业管理后台导航
总览 / 成员与名片 / 企业主页 / 字段与模板 / 数据分析 / 版本与额度 / 管理员 / 审计日志

## 全局 Shell 结构

```
┌ Sidebar(224px, 白底, 右侧1px线) ─┬ Topbar(52px, 面包屑 + 搜索/通知/账号) ┐
│  Logo + 品牌名                    │                                     │
│  一级导航(纯文字, 无图标,          ├ PageHeader(标题24px + 副标题 +      │
│   当前项浅紫底+品牌色字)          │   单个主操作按钮，右对齐)            │
│  底部账号迷你信息                 ├ Content(内边距24px, 可滚动)         │
└──────────────────────────────────┴─────────────────────────────────────┘
```

- 侧边导航**刻意不用图标**，纯文字 + 背景色区分当前态，降低视觉噪声。
- 每页最多 1 个主操作按钮（原型里体现在「成员与名片」的“同步成员”、「企业主页」的“发布”）。
- 危险操作用红字/红色轻按钮，不用大红实心按钮。

## 页面清单与实现要点

以下 6 个页面已做到可交互程度，其余 10 个页面按同一套组件模式实现即可（原型中已给出结构，供参考）：

### 1. 系统后台 / 总览
- 5 个指标卡（等宽 grid），点击可跳转到对应列表页并带上筛选态。
- 风险队列列表（严重度用左侧圆点区分 success/warning/danger/muted 四级）。
- 近 24 小时回调趋势：简单柱状图（成功/失败两色），用真实时序数据替换。
- 快捷入口 4 个，点击跳转对应二级导航。

### 2. 系统后台 / 企业授权中心（企业列表 + 详情抽屉）
- 工具条固定顺序：搜索 → 状态筛选 → （空）→ 刷新，与文档 4.2 一致。
- 表格 8 列：企业名称/OpenCorpID(等宽字体)/授权状态(tag)/授权健康(tag)/成员/名片/安装时间/操作。
- 点击「查看详情」打开右侧 560px 抽屉，5 个分区：授权状态、安全凭据（仅显示是否配置/到期时间，不显示明文）、企业规模（3 个小卡片）、最近回调（列表）、运维动作（重新同步 + 重试失败事件）。
- **重新同步**是危险性中等的操作 → 点击先弹出二次确认 Modal，确认后按钮进入 loading（图标旋转），完成后 toast 提示，全程按钮禁用防重复提交。

### 3. 企业后台 / 总览
- 健康状态 4 个小 chip（授权/成员同步/版本额度/资料完整度），用 tag 颜色表达状态而非纯装饰。
- 4 个指标卡 + 待办列表（每条待办右侧一个跳转链接) + 近 7 日访问趋势柱状图。

### 4. 企业后台 / 成员与名片（列表 + 编辑抽屉）
- 工具条：搜索 + 状态筛选 + 部门筛选。
- 表格 7 列，含脱敏后的手机/邮箱字段（真实脱敏规则由后端返回，前端不做脱敏）。
- 编辑抽屉字段：姓名/部门只读（来自企业微信同步，禁用态样式：浅灰底+灰字）；职位必填，失焦/提交校验；名片状态用开关组件；名片模板下拉在「企业规则禁止编辑」时禁用并显示原因文案；分享码只读+复制按钮。
- **脏数据保护**：抽屉内有未保存修改时，切换到其他成员或点击关闭，先弹出「有未保存的修改」二次确认，选择放弃才真正切换/关闭。保存前校验必填项，报错态紧贴字段显示。

### 5. 企业后台 / 企业主页（三栏编辑器）
- 左栏（160px）：模块列表（基础资料/服务/介绍/荣誉/视频），当前模块高亮。
- 中栏：随模块切换的表单（文本输入/标签增删/多行文本/列表/锁定态说明），视频模块在能力未开通时展示锁定文案，不放可编辑表单。
- 右栏（320px）：手机名片卡样式的**实时预览**，中栏任意字段变化应实时反映到预览（受控组件双向绑定，不要用防抖或提交后才更新）。
- 顶部资料完整度进度条 + 缺失项文案 + 「发布」主操作按钮。

### 6. 企业后台 / 版本与额度
- 当前版本卡（名称/有效期/状态 tag/续费按钮）+ 额度进度卡（进度条颜色随使用率变化，>85% 转警告色）。
- 用量趋势图 + 7天/30天 Segmented 切换。
- 订单表格（订单号建议脱敏展示，状态 tag：已支付=success/处理中=warning/失败=danger）。

### 其余页面（按原型内结构直接复用组件即可）
授权与回调（事件列表+统计卡+重试按钮）、商业化（套餐目录+订单异常队列+人工调整弹窗——**弹窗需要企业选择、方向单选、次数、原因必填、幂等键展示、二次确认**）、功能开关（表格+开关组件）、运维（健康检查卡+迁移任务表）、审计（筛选条+日志表，脱敏摘要不显示明文 diff）、系统账号（账号表+启用/禁用操作）、字段与模板（三个开关的字段规则表+模板卡片+设为默认按钮，当前默认态按钮禁用)、数据分析（指标卡+排行表+转化漏斗进度条)、管理员（角色 tag+移除操作）、审计日志（同系统后台审计的轻量版）。

## 登录页（Admin Login.dc.html）

- 左右分栏：左侧深色科技感视觉区（flex:1.15），右侧浅色表单区（flex:1）。
- 背景：极简深蓝渐变 + 22 个节点、35 条连线组成的网状图，节点呼吸动画 + 数据流动虚线动画；**鼠标悬停节点有弹性放大反馈**（`transition + cubic-bezier(.34,1.56,.64,1)`，实现"划过的弹力感"）。
- 视觉主体：矢量重绘的云朵（渐变层次）、名片主卡片（蓝色头像面板+文字条，渐变+投影提升立体感）、纸飞机、双人像、3D 风格增长图表，均带独立缓慢浮动动画，用虚线+绿色圆点彼此连接，呼应"云端连接、协作、增长"的产品语义。
- 表单：默认选中「企业管理员」Tab（系统管理员在其后）；账号/密码输入、密码显隐切换、记住我勾选、必填校验、提交 loading 态、错误提示。
- 该页应作为独立路由（如 `/login`），登录成功后按账号类型跳转系统后台或企业后台首页。

## 交互规范摘要（详见 original-architecture-guide.md 第 7 节）

- 主操作每个区块最多一个；危险操作用红色文字/轻按钮，不用大红实心按钮做默认态。
- 长任务按钮 loading 时禁用，防止重复提交（重新同步、登录提交均已在原型中体现）。
- 表格：搜索回车触发，筛选变化可自动刷新；空态需给出下一步动作。
- 抽屉优先用于编辑详情；高风险确认用弹窗；弹窗按钮顺序固定为「取消在左，确认在右」。
- 所有弹窗/抽屉在切换侧边栏导航或切换系统/企业后台时应自动关闭，避免遗留在页面上挡住后续操作（这是原型迭代中修复过的一个真实 bug，请在实现路由切换/全局状态重置时留意）。

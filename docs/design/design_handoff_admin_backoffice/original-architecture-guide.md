# 管理后台界面架构设计文档

状态：重构设计基线  
制定日期：2026-07-16  
关联后端指南：[`../01-specs/01_08_Admin_Backoffice_Architecture_Guide.md`](../01-specs/01_08_Admin_Backoffice_Architecture_Guide.md)

## 1. 设计目标

新版管理后台要从“联调控制台”升级为现代 SaaS 管理系统。视觉方向是现代、简洁、高端、交互友好、视觉舒适，不做营销页，不做浮夸装饰，不用大面积单色渐变。第一屏直接进入工作台，帮助管理员快速判断状态、定位问题、完成操作。

关键词：

- 清爽：低噪声、强层级、少装饰。
- 高端：克制色彩、细腻留白、稳定网格。
- 高效：列表密度适中，关键操作不绕路。
- 安全：权限不可见、危险操作二次确认、敏感信息默认脱敏。
- 可扩展：系统管理员和企业管理员共用设计语言，但导航和内容域清晰分离。

## 2. 用户与场景

| 用户 | 目标 | 设计重点 |
|---|---|---|
| 系统管理员 | 管理企业授权、套餐、订单、功能开关、迁移、异常 | 风险看板、跨企业检索、强审计、问题队列 |
| 企业 owner | 管理企业配置、管理员、版本额度 | 全局总览、关键设置、购买续费入口 |
| 企业 admin | 管理成员、名片、模板、字段、企业主页 | 高效编辑、批量状态、清晰保存反馈 |
| 企业 operator | 日常维护成员名片 | 简化导航、减少危险入口 |
| 企业 auditor | 查看数据和日志 | 只读状态明确，导出/筛选友好 |

## 3. 产品信息架构

### 3.1 双后台框架

登录成功后根据 `account_type` 决定后台模式：

```text
session/me
  -> account_type=platform -> 系统管理后台
  -> account_type=tenant   -> 企业管理后台
```

后台不提供“自由切换身份”的入口。平台管理员需要查看企业上下文时，进入企业详情的只读运维视图，不伪装成该企业管理员。

### 3.2 系统管理后台导航

| 一级导航 | 页面 | 主要内容 |
|---|---|---|
| 总览 | `/admin/platform/dashboard` | 授权健康、订单异常、额度风险、回调失败、迁移状态 |
| 企业 | `/admin/platform/tenants` | 企业列表、授权详情、成员/名片规模、同步状态 |
| 授权与回调 | `/admin/platform/wecom` | suite ticket、授权回调、数据回调、失败重试 |
| 商业化 | `/admin/platform/commercial` | 套餐、版本映射、订单、订阅、额度账本、调整 |
| 功能开关 | `/admin/platform/features` | 平台默认、企业 override、视频能力 |
| 运维 | `/admin/platform/ops` | 健康检查、数据库迁移、指标、任务 |
| 审计 | `/admin/platform/audit` | 平台和企业管理操作日志 |
| 系统账号 | `/admin/platform/accounts` | 平台账号、角色、MFA、禁用 |

### 3.3 企业管理后台导航

| 一级导航 | 页面 | 主要内容 |
|---|---|---|
| 总览 | `/admin/tenant/dashboard` | 企业状态、成员名片、访问趋势、版本额度、待办 |
| 成员与名片 | `/admin/tenant/members` | 成员列表、同步、名片编辑、状态、分享码 |
| 企业主页 | `/admin/tenant/company` | 企业资料、服务、介绍、荣誉、视频 |
| 字段与模板 | `/admin/tenant/design` | 字段规则、模板、默认样式、品牌色 |
| 数据分析 | `/admin/tenant/analytics` | 访问趋势、成员排行、互动转化 |
| 版本与额度 | `/admin/tenant/billing` | 当前版本、订单、额度、用量、购买入口 |
| 管理员 | `/admin/tenant/admins` | 管理员列表、角色、owner 认领 |
| 审计日志 | `/admin/tenant/audit` | 本企业操作记录 |

## 4. 页面布局体系

### 4.1 全局布局

桌面端：

```text
┌───────────────┬──────────────────────────────────────────┐
│ Sidebar       │ Topbar: title, breadcrumbs, account       │
│               ├──────────────────────────────────────────┤
│ Navigation    │ Page header: status, primary action       │
│               ├──────────────────────────────────────────┤
│               │ Content: metrics, tables, editors         │
└───────────────┴──────────────────────────────────────────┘
```

移动端和平板：

1. Sidebar 收起为顶部菜单。
2. 列表操作进入底部抽屉或全屏详情。
3. 表格变为摘要列表，保留搜索、筛选、主要操作。

### 4.2 内容区规则

1. 页面使用全宽背景和约束内容宽度，不把大区块层层包成卡片。
2. 卡片只用于指标、列表项、详情片段、弹窗和编辑器。
3. 每个页面只有一个主操作按钮，例如“同步成员”“保存配置”“新增模板”。
4. 危险操作使用次级样式加确认，不与主操作并排抢视觉。
5. 表格工具条固定顺序：搜索、筛选、批量动作、刷新。

## 5. 视觉设计系统

### 5.1 色彩

建议基调：

| Token | 用途 | 建议值 |
|---|---|---|
| `--color-brand` | 主按钮、选中态 | `#2563EB` |
| `--color-accent` | 辅助高亮 | `#0F766E` |
| `--color-bg` | 页面背景 | `#F6F7F9` |
| `--color-surface` | 面板背景 | `#FFFFFF` |
| `--color-text` | 主文本 | `#111827` |
| `--color-muted` | 次文本 | `#667085` |
| `--color-line` | 边框 | `#E5E7EB` |
| `--color-success` | 成功 | `#16A34A` |
| `--color-warning` | 警告 | `#D97706` |
| `--color-danger` | 危险 | `#DC2626` |

使用规则：

1. 主色只用于强调，不铺满页面。
2. 状态色只表达状态，不用于普通装饰。
3. 避免大面积紫蓝渐变、深蓝整页、米色整页等单一主题。
4. 列表状态使用浅底色标签，标签文字必须有足够对比。

### 5.2 字体与密度

| 元素 | 桌面端 |
|---|---|
| 页面标题 | 24-28px |
| 区块标题 | 16-18px |
| 正文 | 14px |
| 表格 | 13-14px |
| 辅助文本 | 12-13px |

规则：

1. 字号不随视口宽度缩放。
2. 字距保持 0。
3. 按钮、标签、表头不使用过宽字距。
4. 后台界面优先信息密度，不使用超大 hero 字体。

### 5.3 圆角、阴影和边框

1. 常规控件圆角 6px。
2. 卡片圆角 8px。
3. 大弹窗圆角 10px。
4. 阴影轻量，只用于浮层、抽屉、顶部悬浮条。
5. 表格和编辑区优先用边框与背景层次，不依赖重阴影。

## 6. 核心页面设计

### 6.1 系统总览

首屏内容：

1. 授权企业数、异常授权数、今日回调失败、订单待处理、额度风险企业。
2. 风险队列：授权异常、订单异常、通知失败、迁移待执行。
3. 最近 24 小时回调趋势。
4. 快捷入口：企业授权、异常订单、数据库迁移、功能开关。

交互：

- 指标点击进入带筛选的列表。
- 风险队列按严重度排序。
- 不展示 token、secret 或敏感企业数据。

### 6.2 企业授权中心

列表字段：

| 字段 | 说明 |
|---|---|
| 企业名称 | 主字段 |
| OpenCorpID | 小号等宽显示，可复制 |
| 授权状态 | 标签 |
| 授权健康 | 正常/需检查 |
| 成员 | 活跃/总数 |
| 名片 | 启用/总数 |
| 安装时间 | 时间 |
| 操作 | 查看详情 |

详情页分区：

1. 授权状态：状态、安装、取消、AgentID。
2. 安全凭据：只显示是否已配置、是否缓存、到期时间。
3. 企业规模：成员、管理员、名片。
4. 最近回调：类型、状态、时间、错误摘要。
5. 运维动作：重新同步、重试失败事件，按权限显示。

### 6.3 企业总览

首屏内容：

1. 企业健康状态：授权、成员同步、版本额度、资料完整度。
2. 四个关键指标：成员、启用名片、近 7 日访问、剩余额度。
3. 待办：未发布企业主页、额度低、同步失败、字段未配置。
4. 近 7 日访问趋势。

企业管理员看到的是行动建议，企业审计员看到的是只读状态。

### 6.4 成员与名片

布局：

1. 左侧成员列表或全宽表格。
2. 右侧抽屉编辑成员名片。
3. 顶部固定搜索、状态、部门、同步按钮。

表格字段：

- 姓名
- 职位
- 手机/邮箱脱敏
- 名片状态
- 最近同步
- 最近访问
- 操作

编辑体验：

1. 保存按钮固定在抽屉底部。
2. 字段被企业规则禁用时显示只读原因。
3. 保存后给出轻提示并刷新当前行。
4. 切换成员时如果表单脏数据，先确认。

### 6.5 企业主页

采用分栏编辑：

1. 左侧模块列表：基础资料、服务、介绍、荣誉、视频。
2. 中间编辑表单。
3. 右侧实时预览，桌面端显示，小屏放到独立预览页。

规则：

- 视频能力未开通时显示锁定状态和联系平台提示。
- 图片 URL、视频 ID、荣誉排序即时校验。
- 发布前显示资料完整度和缺失项。

### 6.6 字段与模板

字段规则：

1. 表格列为字段名、是否启用、员工是否可编辑、默认可见、说明。
2. 使用开关控件，不用文字按钮表达二元状态。
3. 禁用字段需二次确认，因为会影响员工端展示。

模板管理：

1. 模板列表使用缩略预览、名称、状态、默认标识。
2. 品牌色使用色块和输入框。
3. 设置默认使用明确按钮，当前默认模板不可重复点击。

### 6.7 版本与额度

企业侧页面：

1. 当前版本卡：版本、有效期、状态、购买入口。
2. 额度进度：总额度、已用、剩余、使用率。
3. 用量趋势：近 7/30 天。
4. 订单列表：订单号掩码、版本、金额、状态、时间。
5. 消费明细：按日、成员、名片聚合，不展示访客身份。

平台侧页面：

1. 套餐目录和企业微信版本 ID 映射。
2. 订单异常队列。
3. 订阅详情。
4. 额度账本。
5. 人工调整弹窗。

人工调整弹窗必须包含：

- 企业
- 调整方向
- 次数
- 原因
- 幂等键自动生成
- 风险提示
- 二次确认

### 6.8 审计日志

筛选项：

- 时间范围
- 操作者
- 角色
- 动作
- 目标类型
- 风险级别

列表字段：

- 时间
- 操作者
- 身份域
- 动作
- 目标
- 结果
- IP/请求 ID

详情只显示脱敏摘要和 hash，不显示敏感 before/after 明文。

## 7. 交互规范

### 7.1 按钮

1. 主操作每个区块最多一个。
2. 图标按钮用于刷新、复制、筛选、更多、关闭，必须有 tooltip。
3. 危险操作使用红色文字或红色轻按钮，不使用大红实心按钮做默认操作。
4. 长任务按钮进入 loading，禁止重复提交。

### 7.2 表单

1. 必填项在提交前和失焦时校验。
2. 错误提示贴近字段。
3. 保存成功不跳走，除非创建类动作进入详情。
4. 敏感字段默认掩码，复制操作记录审计。

### 7.3 表格

1. 后台核心列表默认 20 条分页。
2. 搜索输入回车触发，筛选改变可自动刷新。
3. 空状态给出下一步动作，例如“同步成员”或“清除筛选”。
4. 加载状态使用骨架或行内 loading，不让页面整体跳动。

### 7.4 弹窗与抽屉

1. 编辑详情优先抽屉。
2. 高风险确认使用弹窗。
3. 强权限操作弹窗必须要求原因。
4. 弹窗按钮顺序：取消在左，确认在右。

## 8. 权限可见性

前端根据 `session/me` 返回的 `permissions` 和 `menu_scopes` 渲染导航：

```json
{
  "admin": {
    "account_type": "tenant",
    "role": "admin",
    "permissions": ["tenant.member.sync", "tenant.config.write"],
    "menu_scopes": ["tenant.dashboard", "tenant.members", "tenant.company"]
  }
}
```

规则：

1. 没权限的一级菜单不显示。
2. 没权限的写按钮不显示。
3. 只读角色看到禁用态时必须知道原因，例如“只读角色不可编辑”。
4. 即使前端隐藏，后端仍必须校验权限。

## 9. 状态与文案

状态文案使用业务语言，不暴露内部异常字符串：

| 后端状态 | 前端文案 |
|---|---|
| `active` | 正常 |
| `cancelled` | 已取消授权 |
| `quota_exhausted` | 展示额度已用尽 |
| `subscription_expired` | 版本已到期 |
| `retryable_failed` | 可重试失败 |
| `manual_review` | 需人工处理 |

公开访客不可用态不得出现“余额不足”“企业未付费”等措辞。后台可以对企业管理员说明额度和版本原因。

## 10. 响应式设计

断点建议：

| 断点 | 处理 |
|---|---|
| `>=1200px` | 固定侧边栏，内容双栏/三栏 |
| `900-1199px` | 侧边栏收窄，详情抽屉宽度 520-640px |
| `<900px` | 顶部导航，表格转列表，详情全屏 |

所有固定格式元素需要稳定尺寸：

1. 图标按钮 36x36。
2. 状态标签最小高度 24。
3. 指标卡高度一致。
4. 表格操作列不因按钮 loading 改变宽度。

## 11. 前端实现建议

当前 `admin/` 是原生 HTML/CSS/JS 静态后台。重构建议进入：

```text
admin-next/
  src/
    app/
    routes/
    components/
    features/
      platform/
      tenant/
      commercial/
      audit/
    api/
    styles/
```

推荐栈与项目文档保持一致：

- React
- TypeScript
- Vite
- Ant Design 或同等成熟组件库
- TanStack Query 管理服务端状态

组件分层：

| 层 | 内容 |
|---|---|
| `api/` | 后台 API client、schema、错误映射 |
| `features/` | 领域页面和 hooks |
| `components/` | Shell、TableToolbar、StatusTag、MetricCard、ConfirmDialog |
| `styles/` | tokens、布局、状态色 |

旧 `admin/` 可继续作为 dev-only 联调控制台，生产入口切换到新版构建产物。

## 12. 可用性验收清单

- [ ] 系统管理员和企业管理员登录后看到完全不同的导航。
- [ ] 企业 auditor 看不到写按钮，直接请求写接口返回 403。
- [ ] 平台详情不显示 token、secret、永久授权码明文。
- [ ] 企业版本与额度页能解释当前状态和下一步。
- [ ] 额度耗尽时，企业后台可见原因，公开访客只见通用不可用态。
- [ ] 表格在移动端不横向挤爆，核心信息转为列表。
- [ ] 所有危险操作都有确认、原因和 loading 防重。
- [ ] 页面中不存在大面积装饰性渐变、嵌套卡片、文字溢出和控件跳动。

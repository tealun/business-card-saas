# Security Policy

## Supported Versions

This repository is maintained as an active SaaS codebase rather than a multi-release
product line. Security fixes are applied to the current `main` branch and the
deployed code behind the repository's backend/admin workflows. Older tags and
historic branches are not security-supported unless they are explicitly called out
in a later release policy.

| Version | Supported |
| ------- | --------- |
| `main` | :white_check_mark: |
| older tags / archived branches | :x: |

When the project starts publishing tagged releases, this table should be updated
to list the supported release lines explicitly.

## Reporting a Vulnerability

Please report security issues privately through GitHub Security Advisories for
this repository.

Do not open a public issue for a vulnerability unless a maintainer asks you to.

Include:

- the affected area or URL;
- the impact you observed;
- steps to reproduce, if available;
- any suggested workaround.

Security reports are triaged by the maintainers before any public discussion.
If the report is accepted, we will confirm receipt and coordinate a fix timeline.
If it is declined, we will explain the reason at a high level without exposing
attack details.
